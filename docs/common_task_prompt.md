# タスク
以下のソースコード群を分析し、Claude Codeが効率的に理解・作業できるドキュメントを生成してください。

# 目的
- トークン消費を最小化しつつ、コード全体を読んだのと同等の理解を得られるドキュメント
- 将来の修正・拡張時にこのドキュメントだけ読めば作業できる状態

# 出力形式（FEATURE.md）

## 概要
この機能が何をするか1-2文で

## ファイル構成
```
ファイル名: 一言説明
```

## クラス構成図
```
親クラス
  └─ 子クラス（役割）
```

## 主要クラス詳細
各クラスについて:
- 責務: 何をするクラスか
- 主要メンバ変数: 名前と用途
- 主要メソッド: 名前、引数、戻り値、処理概要
- 依存関係: 他のどのクラスを使うか

## データフロー
入力 → 処理A → 処理B → 出力 の流れ

## 命名規則・コーディング規約
このコードで使われているパターン

## 定数・設定値
重要なdefineやconst

## 注意点・トラップ
ハマりやすいポイント、暗黙の前提

## 拡張ポイント
機能追加する場合どこを触るか

# 制約
- 冗長な説明は避け、箇条書きを活用
- コード片の引用は最小限（シグネチャのみ）
- 実装詳細より「何をしているか」「なぜそうしているか」を重視
